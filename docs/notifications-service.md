# Notifications Service

Сервис уведомлений о сообщениях.

## Функциональность

- **Создание уведомления** — оповещение другого пользователя о сообщении
- **Получение списка уведомлений** — получение ID сообщений, на которые пользователь был уведомлён

## API

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | /v1/channel/notification/new | Создать уведомление |
| POST | /v1/channel/notification/list | Получить список уведомлений |

## Схема данных

- **V1ChannelNotificationNewRequest** — содержит current_user, channel_id, message_id, other_user_login
- **V1ChannelNotificationListResponse** — массив message IDs

## Потенциальные проблемы

- other_user_login опционален — непонятно как иначе указать получателя
- Ответ на создание пустой `{}`
- Неясно: уведомления в рамках канала или глобальные
- Нет механизма "прочитано/не прочитано"
