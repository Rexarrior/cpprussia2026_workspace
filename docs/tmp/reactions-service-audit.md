## Audit Report: reactions-service

### Issues Found

#### 1. Missing Response Schema
**Severity:** critical  
**Description:** В YAML-файле определена только схема запроса `V1LikeTriggerRequest`, но полностью отсутствует схема ответа (response). Невозможно понять:
- Какие данные возвращает эндпоинт после успешного toggle
- Формат ошибок
- Структуру данных о текущих реакциях

#### 2. Неясен механизм работы idempotency_token для remove
**Severity:** major  
**Description:** `idempotency_token` описан как required поле, но документация не объясняет:
- Как токен используется при **удалении** реакции (remove)
- Должен ли клиент отправлять тот же токен что и при добавлении
- Что происходит при коллизии токенов для разных операций (add vs remove)

#### 3. Нецензурное название анимации "shit"
**Severity:** major  
**Description:** В enum анимаций присутствует значение `shit`. Это:
- Нецензурная лексика (даже в английском)
- Странный выбор для production-сервиса
- Может вызвать проблемы с модерацией контента

**Enum values:** like, dislike, heart, **shit**, okay, LOL, smile

#### 4. Отсутствует API для получения реакций на сообщение
**Severity:** critical  
**Description:** Нет эндпоинта для получения списка реакций на конкретное сообщение. Клиент не может:
- Узнать текущие реакции после toggle
- Отобразить количество реакций
- Реализовать UI с отображением реакций других пользователей

#### 5. Toggle не возвращает состояние после операции
**Severity:** major  
**Description:** После вызова `/v1/like/trigger` сервис не возвращает:
- Новое состояние реакции (добавлена или удалена)
- Список текущих реакций пользователя на сообщении
- Количество реакций каждого типа

#### 6. Неполное описание в markdown
**Severity:** minor  
**Description:** В файле `reactions-service.md`:
- Не описан формат ответа
- Не указаны возможные HTTP коды ошибок
- Не описан процесс авторизации

---

### Recommendations

1. **Добавить схему ответа (V1LikeTriggerResponse):**
   - `action`: string (enum: "added" | "removed")
   - `reactions`: array - список всех реакций на сообщение
   - `user_reaction`: string | null - текущая реакция пользователя

2. **Уточнить документацию idempotency_token:**
   - Описать алгоритм определения add vs remove по токену
   - Добавить схему для error response при дублирующем токене

3. **Заменить "shit" на подходящее значение:**
   - Рекомендация: `fire` (огонь), `rocket`, `thinking`, `surprised`

4. **Добавить новый эндпоинт:**
   - `GET /v1/like/{channel_id}/{message_id}` - получить реакции на сообщение

5. **Документировать HTTP status codes:**
   - 200 - успех
   - 400 - некорректный запрос
   - 401 - неавторизован
   - 404 - сообщение не найдено
   - 409 - конфликт (например, duplicate idempotency_token с разной операцией)

6. **Добавить в markdown:**
   - Примеры request/response
   - Описание потенциальных ошибок
